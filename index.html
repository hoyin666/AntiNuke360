介紹

AntiNuke360 是一個強大的 Discord 伺服器防護機器人，專門針對核打擊 (Nuke) 攻擊與被盜帳號行為進行防禦。自 v1.3.0 起，核心資料與伺服器快照全面改用 MySQL 儲存，更安全與穩定。

完全開源、完全免費、100%透明。

功能

自動反 Nuke 防護

持續監控大量刪除頻道、踢出成員等異常行為，在 10 秒內自動識別並阻止攻擊。

伺服器快照系統（MySQL）v1.3.0

自動建立伺服器結構快照，遭受攻擊後可完全還原。快照 JSON 儲存在 MySQL 的 snapshots 資料表中，保留 72 小時。

三層白名單系統

防踢白名單、臨時白名單、永久白名單，區分管理員和擁有者權限，防止濫用。資料儲存在 server_whitelist 表。

全域黑名單系統

由開發者維護的全域黑名單，記錄已知惡意機器人與攻擊者，新成員加入時即時比對並強制移除。

全域白名單系統

由開發者維護的全域白名單，允許指定機器人在所有伺服器中豁免防護。

稽核日誌監控

監控頻道、角色、成員和 Webhook 的建立與刪除，追蹤每一次敏感操作。

權限錯誤監控

自動偵測權限不足情況，當問題累積會通知伺服器擁有者，必要時自動離開。

成員加入掃描

新成員加入時自動檢查黑名單和白名單，將風險降到最低。

記錄頻道系統

管理員可設定記錄頻道，所有防護事件和白名單操作都會記錄。

Administrator 檢查時刻監控

加入時檢查權限，每小時定期驗證，確保持續完整權限。

延遲初始化v1.2.3

加入伺服器 10 分鐘後才檢查權限，避免無法刪除的身分組污染。

反被盜帳偵測v1.2.2

監控 5 秒內在不同頻道發送相同訊息的異常行為，自動踢出被盜帳號並提供一鍵回服邀請。

日誌頻道私訊提示v1.2.4

當伺服器尚未設定日誌頻道時，重要事件會透過私訊發送，並在訊息中提示「在私訊看到代表未設置日誌頻道」，方便管理員排查設定。

防踢白名單引導v1.2.4

黑名單停權通知中會附帶 /add-server-anti-kick 說明，引導伺服器擁有者將可信任帳號加入防踢白名單，避免誤封。

歡迎頻道自動重試v1.2.4

加入伺服器時若建立歡迎頻道失敗，會每分鐘重試一次，直到建立成功或 Bot 自動離開伺服器。

MySQL 資料儲存v1.3.0

全域黑名單、全域白名單、本地伺服器白名單、guild 資訊與快照全部改用 MySQL 儲存，安全性與穩定性大幅提升。

v1.3.0 核心改進

  • MySQL 全面導入：黑名單、白名單、伺服器白名單、guild 資訊與快照統一存放於 MySQL。
  • 快照進階儲存：伺服器結構快照以 JSON 格式存放於 snapshots 資料表，不再依賴本機檔案。
  • `/check-black` 指令：可快速查詢任意 ID 是否在全域黑名單 / 白名單，方便管理與調試。

v1.2.4 核心改進

  • 日誌頻道私訊註記：當在私訊中收到 AntiNuke360 的重要通知時，會清楚標明「若在私訊看到此訊息，代表伺服器尚未設定日誌頻道」，協助管理員快速定位問題。
  • 黑名單停權引導：掃描並停權黑名單成員時，會於通知中加入 /add-server-anti-kick 說明，避免對可信任帳號造成干擾。
  • 歡迎頻道重試機制：Bot 加入伺服器後若因權限或 Discord 狀態導致歡迎頻道建立失敗，會每分鐘自動重試，直到成功或離開伺服器。

安裝

先決條件

  • Python 3.8 或更高版本
  • discord.py
  • python-dotenv
  • MySQL 或 MariaDB 資料庫（可使用主機商提供的 DB）

安裝步驟

  1. 複製倉庫或下載程式碼。
  2. 建立 .env 檔案並設置 Discord Bot Token 與 MySQL 連線資訊： 
    DISCORD_TOKEN=your_discord_bot_token_here

MYSQL_HOST=your_mysql_host_or_container_name
MYSQL_PORT=3306
MYSQL_USER=your_mysql_user
MYSQL_PASSWORD=your_mysql_password
MYSQL_DB=your_mysql_database
  3. 安裝依賴： 
    pip install -r requirements.txt
  4. （可選）先執行資料轉移/建表腳本，將舊版 JSON 轉入 MySQL。
  5. 啟動機器人 (v1.3.0)： 
    python AntiNuke360_v1.3.py

Discord Bot 設置

  1. 前往 Discord Developer Portal。
  2. 建立新 Application，並在 Bot 分頁建立機器人。
  3. 複製 Bot Token 到 .env 檔案。
  4. 在 OAuth2 的 URL Generator 中勾選 Administrator 權限。
  5. 使用生成的邀請連結將機器人加入伺服器。

管理員指令

/status

查看 AntiNuke360 的當前運行狀態、防護參數與統計，包含是否有可用快照與反被盜帳狀態。

/scan-blacklist

掃描伺服器內所有成員，並停權全域黑名單中的成員。需要管理員權限。若要在本伺服器允許特定黑名單帳號，可請伺服器擁有者使用 /add-server-anti-kick 將其加入防踢白名單。

/restore-snapshot

手動還原伺服器最新的快照。還原所有角色、分類、頻道、權限設定。需要管理員權限，快照有效期為 72 小時。

/add-server-temp [ID]

將機器人或成員加入臨時白名單 (1 小時自動過期)。需要管理員權限。臨時白名單成員在敏感操作上享有較寬鬆的限制。

/remove-server-temp [ID]

從臨時白名單移除機器人或成員。需要管理員權限。

/server-whitelist

查看本伺服器的白名單狀態 (防踢、臨時、永久)，以及臨時白名單的剩餘時間。需要管理員權限。

/set-log-channel [#channel]

設置本伺服器的記錄頻道。所有防護事件都會記錄在此。若未設定，重要事件將以私訊方式傳送並附註說明。需要管理員權限。

/toggle-anti-hijack [on/off]

開啟或關閉反被盜帳功能。預設啟用。需要管理員權限。

伺服器擁有者指令

/add-server-anti-kick [ID]

將機器人或成員加入防踢白名單。允許全域黑名單帳號在本伺服器中保留，但仍會記錄監控。

/remove-server-anti-kick [ID]

從防踢白名單移除機器人或成員。

/add-server-perm [ID]

將機器人或成員加入永久白名單。對所有敏感操作完全豁免。

/remove-server-perm [ID]

從永久白名單移除機器人或成員。

開發者指令

/add-black [ID] [原因]

將機器人加入全域黑名單，並在所有伺服器掃描與停權。

/remove-black [ID]

從全域黑名單移除機器人。

/add-white [ID] [原因]

將機器人加入全域白名單，使其在所有伺服器中豁免。

/remove-white [ID]

從全域白名單移除指定機器人。

/blacklist

查看全域黑名單前 10 筆紀錄。

/whitelist-list

查看全域白名單前 10 筆紀錄。

/scan-all-guilds

在所有伺服器中掃描並停權黑名單成員。

/check-black [ID]v1.3.0

查詢某個 ID 是否在全域黑名單或白名單，會顯示名稱、原因、timestamp 與偵測伺服器（若有）。

常見問題

v1.3.0 新增了什麼？

v1.3.0 為一次架構級更新：

  • 黑名單、白名單、本地伺服器白名單、guild 資訊與伺服器快照全部移至 MySQL 儲存，提升穩定與一致性。
  • 快照改存於 snapshots 資料表，不再使用本機 JSON 檔案。
  • 新增 /check-black 指令，可方便查詢任意 ID 是否在黑/白名單中。

v1.2.4 新增了什麼？

v1.2.4 聚焦於實務使用的細節優化：

  • 所有以私訊送出的重要通知都會附帶「若在私訊看到此訊息，代表伺服器尚未設定日誌頻道」說明。
  • 黑名單掃描與自動停權通知會主動提示可使用 /add-server-anti-kick 避免誤封可信任帳號。
  • 加入伺服器時若歡迎頻道建立失敗，Bot 會每分鐘重試建立，直到成功或離開伺服器。

v1.2.3 新增了什麼？

v1.2.3 新增了 10 分鐘延遲初始化機制。機器人不再加入伺服器後立即檢查權限，而是等待 10 分鐘後才檢查。這給予管理員充足時間來配置身分組和權限，避免生成無法刪除的自動身分組。

為什麼要延遲 10 分鐘檢查？

因為有些管理員為了方便整理身份組，會在加入伺服器前拔掉所有權限，隨後再為機器人增加有 Administrator 權限的身份組。若加入時不拔掉所有權限，機器人身分組就會無法刪除。10 分鐘延遲給予管理員時間來手動配置所需的身分組，讓機器人身分組不需要自動創建。

延遲期間發生攻擊怎麼辦？

這取決於有沒有正確設置權限。即使在延遲期間，只要在攻擊發生前正確設置了權限，伺服器仍可受保護。

反被盜帳功能如何工作？

與其他 Bot 不同，AntiNuke360 不使用特定頻道偵測，而是監控 5 秒內在不同頻道發送相同訊息的行為，這是被盜帳號的真實特徵。這樣可以避免誤傷好奇用戶，提高精準度。

可以關閉反被盜帳功能嗎？

可以。使用 /toggle-anti-hijack off 指令關閉反被盜帳功能。預設是啟用的。

三層白名單是在解決什麼問題？

v1.2 修復了一個重要的安全風險：若只有單一白名單層級，任何管理員都可以讓惡意機器人繞過防護。三層設計將「允許加入、不踢出、完全免疫」清楚分開，並限制關鍵權限只能由伺服器擁有者調整。

快照會保留多久？

每個伺服器的快照保留 72 小時（3 天）。超過 72 小時的快照視為失效，不再允許還原。

是。AntiNuke360 永久免費，亦永久開源，不會收取任何費用。
